# 프로젝트 개발 규칙

## 배포 규칙

### 1. 배포 순서
1. **로컬 개발환경 먼저 배포** (기본)
2. 로컬에서 테스트 및 확인
3. **운영 배포는 명시적 요청이 있을 때만** 진행

### 2. 운영 배포 조건
- 사용자가 명시적으로 "운영 배포", "프로덕션 배포", "실서버 배포" 등을 요청한 경우에만 실행
- 단순히 "배포해줘"라고 하면 로컬 개발환경 배포로 간주
- 운영 배포 전에 반드시 사용자에게 확인 요청

### 3. 배포 스크립트
- 로컬: `./start_backoffice.sh` (백오피스), 로컬 서버 재시작
- 운영: `./deploy.sh` (메인 서버), `./deploy_backoffice.sh` (백오피스)

## DB 마이그레이션 규칙

### 1. 마이그레이션 순서
1. 로컬 DB에 먼저 마이그레이션 실행
2. 로컬에서 테스트 및 확인
3. **운영 DB 마이그레이션은 명시적 요청 시에만** 실행

### 2. 운영 DB 마이그레이션
- 반드시 사용자의 명시적 요청 후 실행
- 마이그레이션 스크립트를 제공하고 사용자가 직접 실행하도록 권장
- 자동 실행 시에는 반드시 사용자에게 확인 요청

## 개발 환경

### 로컬 개발 환경
- DB: localhost:5432 (PostgreSQL)
- 메인 서버: localhost:8080
- 백오피스: localhost:9090

### 운영 환경
- 서버: EC2 (13.209.47.72)
- DB: RDS PostgreSQL
- 도메인: haruinfo.net
